# RQ-044: Stakes Psychology Analysis

## 44.1: Conditions for Effectiveness

### When Do Stakes Work? A Conditional Analysis

Stakes are NOT universally helpful or harmful. Effectiveness depends on **five moderating variables**:

| Variable | Stakes HELP When... | Stakes HARM When... |
|----------|---------------------|---------------------|
| **Habit Difficulty** | Low-to-moderate difficulty; user CAN do it but doesn't | High difficulty; user genuinely struggles with capability |
| **User Autonomy** | User CHOSE the stake voluntarily | Stake feels imposed or expected |
| **Relationship Quality** | High-trust, secure relationship with witness | Fragile, competitive, or judgmental relationship |
| **Stake Severity** | Mild/moderate; "skin in the game" not "life-changing loss" | Severe; creates catastrophic thinking |
| **Failure History** | User has recent success; failure is exception | User has repeated failure; stake confirms "I can't" |

### Empirical Evidence

**Study 1: Charness & Gneezy (2009)** â€” Financial incentives for gym attendance
- Paid participants $25/visit for 4 weeks â†’ attendance increased
- **BUT:** After incentive removal, attendance dropped BELOW baseline
- **Implication:** Financial stakes create dependency, not habit

**Study 2: Rogers et al. (2015)** â€” Commitment devices in health behaviors
- Voluntary commitment contracts increased short-term compliance by 33%
- Long-term (6+ months) effects were negligible
- **Key finding:** Effectiveness correlated with AUTONOMY in choosing the contract

**Study 3: Ariely & Wertenbroch (2002)** â€” Self-imposed vs external deadlines
- Self-imposed deadlines (with self-set consequences) improved performance
- Externally imposed deadlines with same consequences did NOT
- **Implication:** Autonomy in stake-setting is critical

## 44.3: Short-Term vs Long-Term Analysis

### The Compliance-Automaticity Tradeoff

| Phase | With Stakes | Without Stakes |
|-------|-------------|----------------|
| **Week 1-4** | Compliance HIGH (external motivation) | Compliance MODERATE (internal only) |
| **Week 5-12** | Compliance maintained IF stakes continue | Automaticity developing |
| **Post-Stakes** | SHARP DROP (overjustification effect) | Gradual stabilization |

### The Overjustification Effect (Deci, 1971; Lepper et al., 1973)

When external rewards/punishments are applied to intrinsically interesting activities:
1. Brain attributes behavior to EXTERNAL cause
2. Internal motivation is "crowded out"
3. When external cause removed â†’ behavior decreases

**Application to Stakes:**
- Stakes create external attribution ("I'm doing this because of the stake")
- This PREVENTS internal attribution ("I'm doing this because I'm The Writer")
- Stakes may BLOCK identity formation â€” the opposite of The Pact's goal

### When Stakes DON'T Cause Overjustification

Research (Cameron & Pierce, 1994; Eisenberger & Cameron, 1996) shows overjustification is reduced when:
1. Reward is **unexpected** (not contingent on performance)
2. Reward is **informational** (signals competence, not control)
3. Activity was NOT intrinsically interesting initially

**Implication for The Pact:**
- For habits users DON'T enjoy yet, stakes may help bridge the gap
- For habits users ARE building identity around, stakes may harm
- **Solution:** Stage-based approach â€” stakes early (building), remove stakes later (maintenance)

**Confidence: HIGH** â€” Core SDT finding, well-replicated.

   - User controls what witness sees

   - Suggested responses to failure: "Tomorrow's a new day" not "You said you would..."

   - Easy for user to remove stakes without witness knowing
   - Prevents "trapped" feeling in commitment

## 44.7: Recovery Path Design

### The POST-FAILURE Experience

When a user fails publicly (witness sees), the system must provide:

**Immediate (0-60 seconds):**
